# Blog Post Structure

## Standard Template (7 Sections)

### 1. Hook (1-2 paragraphs)
- Specific problem or contrarian take
- Grab attention immediately
- Set up the "why should I care?"

### 2. Context (1-2 paragraphs)
- Why this matters now
- Current relevance and urgency
- Connect to reader's situation

### 3. Framework (main body)
- Actionable methodology or mental model
- Use subheadings (H3) for components
- Include specific examples
- Numbered lists for processes
- Bullets for options/considerations

### 4. Case Study (optional)
- Real-world application
- **Only include if:**
  - Idea file provides a real case study, OR
  - Framed as clearly hypothetical ("Consider a scenario...")
- **Never fabricate** specific companies, revenues, or outcomes

### 5. Implementation
- Step-by-step guide
- Practical next steps
- Prioritised actions where possible

### 6. Pitfalls
- Common mistakes to avoid
- Keep brief and punchy
- 3-5 bullet points typically

### 7. Next Steps / CTA
- Clear action reader can take
- Link to related content
- Newsletter signup where appropriate
- Consultation CTA for bottom-of-funnel content

## Formatting Guidelines

### Paragraphs
- 3-4 sentences maximum
- One idea per paragraph
- White space aids scanning

### Lists
- Use bullets for unordered items
- Use numbers for sequences/processes
- Keep list items parallel in structure

### Emphasis
- **Bold** for key phrases (scannable)
- *Italics* sparingly for emphasis
- `Code` for technical terms, commands, file names

### Code Blocks
- Use for templates, examples, configs
- Include language identifier for syntax highlighting
- Keep examples minimal and focused

## Length Guidelines

| Content Type | Word Count |
|--------------|------------|
| Framework/how-to | 1,500-2,000 |
| Thought leadership | 1,800-2,200 |
| Case study deep-dive | 2,000-2,500 |
| Opinion/hot take | 800-1,200 |

**Default target: 1,500-1,800 words**
